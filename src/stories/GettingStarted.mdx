# 🚀 Guia de Início Rápido - Storybook

Bem-vindo à documentação do Sistema de Gestão! Este Storybook contém a documentação interativa de todos os componentes do projeto.

## 📋 O que você encontrará aqui

### 🎨 Design System
- **Theme**: Paleta de cores, tipografia e estilos do projeto
- **Componentes**: Documentação de todos os componentes reutilizáveis

### 🧩 Componentes Principais

#### GenericDataGrid
Componente base para exibição de dados tabulares usado em todo o sistema.

**Funcionalidades:**
- Paginação automática
- Filtros e ordenação
- Estados de carregamento
- Design responsivo

#### Formulários
Todos os formulários do sistema seguem o mesmo padrão de design e UX.

**Características:**
- Validação em tempo real
- Feedback visual
- Responsividade
- Acessibilidade

## 🛠️ Como usar este Storybook

### 1. Navegação
Use o menu lateral para navegar entre:
- **Documentação**: Guias e informações gerais
- **Design System**: Cores, tipografia e tema
- **Components**: Componentes individuais

### 2. Testando Componentes
Cada componente possui:
- **Canvas**: Visualização interativa
- **Docs**: Documentação automática
- **Controls**: Controles para testar diferentes props

### 3. Variações (Stories)
Cada componente tem múltiplas "stories" que mostram:
- Estado padrão
- Estados de erro
- Diferentes tamanhos
- Casos de uso específicos

## 🔧 Comandos Úteis

```bash
# Iniciar o Storybook
npm run storybook

# Construir o Storybook para produção
npm run build-storybook

# Iniciar a aplicação principal
npm start
```

## 🎯 Boas Práticas

### Para Desenvolvedores
1. **Sempre teste componentes no Storybook** antes de integrar
2. **Documente novos componentes** criando stories
3. **Use o tema consistentemente** em todos os componentes
4. **Teste diferentes estados** (loading, error, empty)

### Para Designers
1. **Use o Design System** como referência
2. **Verifique a consistência** visual entre componentes
3. **Teste a responsividade** em diferentes tamanhos
4. **Valide a acessibilidade** dos componentes
